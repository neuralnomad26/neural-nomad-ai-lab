# Module 02 — Exploratory Data Analysis

> **Status:** ⬜ Upcoming

Before modelling, understand your data. EDA is not a formality — it is where bad assumptions get caught early, where interesting patterns reveal themselves, and where good feature engineering begins.

---

## Learning Objectives

- Characterise any dataset: distributions, central tendency, spread, skewness
- Detect and handle missing values, outliers, and data quality issues
- Produce publication-quality visualisations with Matplotlib, Seaborn, and Plotly
- Form and test data hypotheses before touching a model
- Build a repeatable EDA template that can be applied to any dataset

---

## Topics Covered

| Notebook | Topics |
|----------|--------|
| `01_univariate_analysis.ipynb` | Histograms, KDE plots, box plots, summary stats |
| `02_bivariate_analysis.ipynb` | Scatter plots, correlation matrices, pair plots |
| `03_categorical_analysis.ipynb` | Bar charts, count plots, chi-squared tests |
| `04_missing_values.ipynb` | Patterns of missingness, MCAR/MAR/MNAR, imputation |
| `05_outlier_detection.ipynb` | IQR, Z-score, isolation forest, visual methods |
| `06_feature_distributions.ipynb` | Skewness, log transforms, power transforms |
| `07_time_series_eda.ipynb` | Trends, seasonality, rolling stats |
| `08_eda_report_template.ipynb` | Full reusable EDA pipeline on a real dataset |
